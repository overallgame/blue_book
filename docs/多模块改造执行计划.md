# 多模块改造执行计划

> 基于 `docs/多模块改造方案.md`，细化到每个文件的操作步骤。

---

## 总体原则

- **每完成一个模块的迁移，立即编译验证**（`./gradlew assembleDebug`），不攒到最后
- **先迁文件，再删文件**：新模块创建 → 迁入代码 → `:app` 添加依赖 → 编译通过 → 删除 `:app` 旧文件
- **一次只拆一个模块**，不并行拆多个，出问题容易定位
- **每个模块迁移完毕后都提交一次git**，每个模块迁移完毕后，通过编译，删除`:app` 旧文件，提交git

**3. 新建 `SessionDataStore.kt`**：
```kotlin
package com.example.blue_book.datastore

import android.content.Context
import androidx.datastore.core.DataStore
import androidx.datastore.preferences.core.Preferences
import androidx.datastore.preferences.core.edit
import androidx.datastore.preferences.core.stringPreferencesKey
import androidx.datastore.preferences.preferencesDataStore
import dagger.hilt.android.qualifiers.ApplicationContext
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.map
import javax.inject.Inject
import javax.inject.Singleton

private val Context.dataStore: DataStore<Preferences> by preferencesDataStore(name = "session")

@Singleton
class SessionDataStore @Inject constructor(
    @ApplicationContext private val context: Context
) {
    companion object {
        private val KEY_PHONE = stringPreferencesKey("phone")
        private val KEY_AUTH_TOKEN = stringPreferencesKey("auth_token")
        private val KEY_REFRESH_TOKEN = stringPreferencesKey("refresh_token")
    }

    val phone: Flow<String?> = context.dataStore.data.map { it[KEY_PHONE] }
    val authToken: Flow<String?> = context.dataStore.data.map { it[KEY_AUTH_TOKEN] }
    val refreshToken: Flow<String?> = context.dataStore.data.map { it[KEY_REFRESH_TOKEN] }

    suspend fun saveTokens(phone: String, authToken: String, refreshToken: String) {
        context.dataStore.edit { prefs ->
            prefs[KEY_PHONE] = phone
            prefs[KEY_AUTH_TOKEN] = authToken
            prefs[KEY_REFRESH_TOKEN] = refreshToken
        }
    }

    suspend fun savePhone(phone: String) {
        context.dataStore.edit { it[KEY_PHONE] = phone }
    }

    suspend fun clear() {
        context.dataStore.edit { it.clear() }
    }
}
```

**处理 AuthInterceptor 的 token 读取**：
```kotlin
// AuthInterceptor 中
// 旧：preferences.getAuthToken()
// 新：runBlocking { sessionDataStore.authToken.first() ?: "" }
// 或用 CoroutineScope.launch 异步读，但 Interceptor.intercept() 是同步方法
// 推荐：保留一个小 SharedPreferences 仅给 AuthInterceptor 用（不迁移到 DataStore）
```

> **决策**：`AuthInterceptor` 继续使用 `AuthPreferences`（SharedPreferences），不改为 DataStore。原因是 OkHttp Interceptor 的 `intercept()` 方法是同步的，DataStore 是异步的，用 `runBlocking` 会阻塞网络线程。`TokenAuthenticator` 同理。

### 步骤 5.1 — 创建 `:feature-main`

**迁入**：
```
app/...presentation/main/MainTabsFragment.kt → :feature-main/.../main/MainTabsFragment.kt
```

**迁入资源**：
| 原路径 | 目标 |
|--------|------|
| `layout/activity_main.xml` | `:feature-main/res/layout/` |
| `menu/bottom_navigation__view.xml` | `:feature-main/res/menu/` |
| `menu/main_bottom_navigation.xml` | `:feature-main/res/menu/` |
| `drawable/navigation_item_selector.xml`（color + drawable） | `:feature-main/res/` |

**改造 MainTabsFragment**：
```kotlin
// 旧：ViewPager2 adapter 中直接 new Fragment
// 新：通过 ARouter 获取 Fragment 实例
override fun getItem(position: Int): Fragment {
    return when (position) {
        0 -> ARouter.getInstance().build(RouterPath.Home.MAIN).navigation() as Fragment
        1 -> ARouter.getInstance().build(RouterPath.Video.TAB).navigation() as Fragment
        2 -> ARouter.getInstance().build(RouterPath.Message.MAIN).navigation() as Fragment
        3 -> ARouter.getInstance().build(RouterPath.Mine.MAIN).navigation() as Fragment
        else -> throw IllegalArgumentException()
    }
}
```

**依赖**：`:lib-base`、ARouter（不需要依赖具体 feature 模块，ARouter 运行时发现路由）
