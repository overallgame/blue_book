# 多模块改造方案

> 版本：v1.0 | 日期：2026-06-03

---

## 一、改造背景与目标

### 1.1 当前问题

| 问题 | 现状 |
|------|------|
| 单一模块 | 所有代码集中在 `:app`，编译速度随代码量线性下降 |
| 耦合风险 | 跨 feature 的类可以随意 import，边界模糊 |
| 编译粒度大 | 改一行代码也要全量编译 debug APK |
| 无法并行构建 | 单模块无法利用 Gradle 并行编译能力 |
| 跨模块通信缺失 | 页面跳转靠 Navigation XML 硬编码路径，跨 feature 通信靠手动回调 |
| 本地存储分散 | SharedPreferences（`AuthPreferencesImpl`）不支持协程，无类型安全 |

### 1.2 改造目标

1. **16 个模块**的 Clean Architecture 多模块结构
2. **ARouter** 替代 Jetpack Navigation 做跨模块页面路由
3. **EventBus** 做跨模块事件通信（登录态变更、点赞收藏同步等）
4. **DataStore** 替代 SharedPreferences 做本地持久化
5. 每个 feature 模块可独立编译，支持 Gradle 并行构建

---

## 二、引入的第三方库

### 2.1 ARouter（页面路由）

| 项目 | 说明 |
|------|------|
| 仓库 | `com.alibaba:arouter-api` + `arouter-compiler` |
| 用途 | 替代 `nav_graph.xml`，实现跨模块页面跳转 |
| 选型理由 | 国内 Android 项目最成熟的路由框架，支持拦截器、降级、依赖注入 |
| 注解 | `@Route(path = "/feature/xxx")` 注册路由，`@Autowired` 自动注入参数 |

### 2.2 EventBus（事件总线）

| 项目 | 说明 |
|------|------|
| 仓库 | `org.greenrobot:eventbus` |
| 用途 | 跨模块事件通知：登录/登出、头像更新、点赞/收藏状态同步 |
| 选型理由 | 轻量，Sticky 事件支持（后注册的订阅者也能收到上一次事件），适合跨模块解耦 |
| 注解 | `@Subscribe` 注册订阅方法，`EventBus.post()` 发送事件 |

### 2.3 DataStore（本地持久化）

| 项目 | 说明 |
|------|------|
| 仓库 | `androidx.datastore:datastore-preferences` |
| 用途 | 替代 `SharedPreferences`（`AuthPreferencesImpl`），存储 token/phone 等键值对 |
| 选型理由 | 协程原生支持，`Flow<T>` 自动通知数据变化，类型安全，不受 ANR 影响 |

## 四、各模块详细设计

### 4.1 `:lib-base` — 基础库（Android Library）

> **重要变更**：`:lib-base` 必须为 **Android Library** 模块（`com.android.library`），而非纯 Kotlin 模块。原因：
> 1. `VideoCardInfo` 使用 `@Parcelize` 需要 Android SDK
> 2. `UdfViewModel` 继承 `androidx.lifecycle.ViewModel`
> 3. 需要承载**跨模块共享的 ViewBinding 布局、Drawable、values 资源**（如公共 shape、占位图、点赞图标等）

**依赖**：无（仅 `kotlin-stdlib`、`androidx-core`、`lifecycle-viewmodel`、`coroutines`）

**移入的文件**：
```

common/util/UriFileResolver.kt          → 改为接口，Android 实现迁到 :core-datastore
core/udf/UdfViewModel.kt
core/udf/UiIntent.kt
core/udf/UiState.kt

```

**新增内容**：
```

event/LoginEvent.kt        ← 登录事件
```

### 4.4 `:data-local` — 本地存储层

**依赖**：`:domain`、`:lib-base`、`:core-datastore`

**移入的文件**：
```

data/local/db/resource/user/UserLocalDataResourceImpl.kt
```

**改造：SharedPreferences → DataStore**

原 `data/local/preference/AuthPreferences.kt` + `AuthPreferencesImpl.kt` **废弃**。

在 `:core-datastore` 中新建 `SessionDataStore.kt`：
```kotlin
class SessionDataStore @Inject constructor(
    @ApplicationContext private val context: Context
) {
    private val dataStore = context.dataStore

    val phone: Flow<String?> = dataStore.data.map { it[KEY_PHONE] }
    val authToken: Flow<String?> = dataStore.data.map { it[KEY_AUTH_TOKEN] }
    val refreshToken: Flow<String?> = dataStore.data.map { it[KEY_REFRESH_TOKEN] }

    suspend fun saveTokens(phone: String, authToken: String, refreshToken: String) {
        dataStore.edit { prefs ->
            prefs[KEY_PHONE] = phone
            prefs[KEY_AUTH_TOKEN] = authToken
            prefs[KEY_REFRESH_TOKEN] = refreshToken
        }
    }

    suspend fun clear() {
        dataStore.edit { it.clear() }
    }
}
```

**变更影响**：
- `AuthInterceptor` 从读 `AuthPreferences` 改为读 `SessionDataStore`（需适配 Flow → 同步读取）
- `TokenAuthenticator` 的 `clearAuthData()` 改为调用 `SessionDataStore.clear()`
- `UserLocalDataResourceImpl` 改为依赖 `SessionDataStore` + `UserDao`

> **注意**：`AuthInterceptor` 是 `Interceptor` 接口，运行在 OkHttp 线程池，不能直接调 suspend 函数。需要使用 `runBlocking` 读取 DataStore 的一次性快照（`dataStore.data.first()`），或保留小体积的 SharedPreferences 仅用于 Interceptor 场景（更推荐后者，避免阻塞 I/O 线程）。

| `CollectToggleEvent` | videoId, isCollected | 普通 |

### 7.2 事件流向

```
:feature-auth  ──LoginEvent(sticky)──→  :feature-mine（刷新用户信息）
                                      →  :feature-home（可选：刷新首页数据）

:feature-mine  ──LogoutEvent────────→  :feature-auth（回到登录页）

:feature-mine  ──AvatarUpdateEvent──→  :feature-video（评论列表刷新头像）

:feature-video ──LikeToggleEvent────→  :feature-mine（喜欢列表同步）
:feature-home  ──LikeToggleEvent────→  :feature-mine

:feature-video ──CollectToggleEvent─→  :feature-mine（收藏列表同步）
:feature-home  ──CollectToggleEvent─→  :feature-mine
```

### 7.3 使用示例

```kotlin
// 发送方
EventBus.getDefault().postSticky(LoginEvent(phone, nickname, avatar))

// 订阅方（需在 onStart/onCreate 中注册，onStop/onDestroy 中取消）
@Subscribe(threadMode = ThreadMode.MAIN, sticky = true)
fun onLoginEvent(event: LoginEvent) {
    // 更新 UI
}

override fun onStart() {
    super.onStart()
    EventBus.getDefault().register(this)
}

override fun onStop() {
    super.onStop()
    EventBus.getDefault().unregister(this)
}
```

## 十一、风险与注意事项

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Hilt 多模块处理器兼容性 | 编译失败 | 每个模块添加 `kapt { correctErrorTypes = true }`，确保 kapt 正确生成 |
| DataStore 在 OkHttp Interceptor 中读取 | 阻塞 I/O 线程 | `AuthInterceptor` 场景保留轻量 SharedPreferences，其他场景全部迁移 DataStore |
| ARouter 参数传递 Parcelable | 编译期校验 | `VideoCardInfo` 确保 `@Parcelize`，所有跨模块传输的 bean 实现 Parcelable |
| EventBus 注册/反注册遗漏 | 内存泄漏 | 统一在 BaseFragment/BaseActivity 的生命周期回调中注册和反注册 |
| 模块间循环依赖 | 编译失败 | 严格执行依赖层次图，`:lib-base` 和 `:domain` 不能依赖任何 feature 模块 |
| 编译速度初期反而变慢 | 开发效率 | Android Library 模块首次编译慢，但增量编译和并行构建收益会在后续体现 |
