# 数据层拆分方案

> 将集中式 `:data-api`、`:data-repository`、`:domain` 拆散，按业务归属分入各 feature 模块。

---

## 一、拆分原则

- **业务闭环**：每个 feature 模块拥有自己的 API 接口、DTO、Repository、Domain Model、UseCase
- **共享保留**：跨模块共用的基础能力保留在独立共享模块
- **网络层统一**：OkHttp、Retrofit、Gson、RemoteCall 统一在 `:core-network`
- **本地存储统一**：Room、DataStore 保留在 `:data-local`

## 二、模块最终状态

### 删除的模块（3 个）

| 模块 | 原因 |
|------|------|
| `:data-api` | 合并到 `:core-network`（网络基础设施） |
| `:data-repository` | 按业务拆入 feature 模块 |
| `:domain` | 按业务拆入 feature 模块 |

### 保留的模块

```
:lib-base         ← UDF基类 + ApiResponse + CommonResultDto + 共享资源
:core-network     ← OkHttp + Retrofit + Gson + RemoteCall + TokenAuthenticator
:core-player      ← 播放器引擎（不变）
:core-datastore   ← DataStore + TokenCache（不变）
:data-local       ← Room数据库 + UserDao + AuthDataStoreImpl（共享）
:feature-auth     ← AuthActivity + auth domain + auth api + auth repository
:feature-home     ← HomeActivity + search + 瀑布流（无独立api，复用video）
:feature-video    ← VideoActivity + video domain + video api + comment api
:feature-message  ← MessageActivity（占位）
:feature-mine     ← MineActivity + user domain + user api + user repository
:feature-image    ← ImagePickerActivity（不变）
:app              ← MainActivity + 主题（壳）
```

## 三、`:domain` 文件归属

### → `:lib-base`（跨模块共享的模型）

| 文件 | 原因 |
|------|------|
| `domain/model/UserAccount.kt` | :feature-auth 和 :feature-mine 都需要 |
| `domain/model/LoginCredentials.kt` | 同上 |

### → `:feature-auth`（认证相关）

| 类别 | 文件 |
|------|------|
| model | `RegisterInfo.kt` |
| repository | `domain/repository/UserRepository.kt` → 拆出 `AuthRepository.kt`（login/register/logout/sendCode） |
| usecase | `LoginUseCase.kt`, `RegisterUseCase.kt`, `LogoutUseCase.kt`, `IsLoggedInUseCase.kt`, `SendVerificationCodeUseCase.kt` |

### → `:feature-mine`（用户相关）

| 类别 | 文件 |
|------|------|
| repository | `domain/repository/UserRepository.kt` → 拆出 `UserRepository.kt`（profile/update/avatar） |
| usecase | `GetUserProfileUseCase.kt`, `UpdateUserProfileUseCase.kt`, `GetCurrentUserPhoneUseCase.kt` |

### → `:feature-video`（视频+评论）

| 类别 | 文件 |
|------|------|
| model | `Video.kt`, `Comment.kt` |
| repository | `VideoRepository.kt`, `CommentRepository.kt` |
| usecase | `FetchRandomVideosUseCase.kt`, `FetchVideosByKeywordUseCase.kt`, `FetchPlayUrlUseCase.kt`, `LikeVideoUseCase.kt`, `CollectVideoUseCase.kt`, `FetchCommentsUseCase.kt`, `FetchRepliesUseCase.kt`, `PostCommentUseCase.kt`, `ReplyCommentUseCase.kt`, `LikeCommentUseCase.kt`, `DeleteCommentUseCase.kt`（11 个） |

## 四、`:data-api` 文件归属

`:data-api` 所有内容**合并到 `:core-network`**（作为统一的网络数据层）：

| 内容 | 说明 |
|------|------|
| `RemoteCall.kt` | 迁入 `:core-network/data/remote/`（同包名） |
| 6 个 `*Api.kt` 接口 | 迁入 `:core-network/data/remote/` |
| 所有 DTO（28 个） | 迁入 `:core-network/data/remote/` |
| 6 个 `*RemoteDataSource.kt` | 迁入 `:core-network/data/remote/` |
| `ApiModule.kt` | 内容合并到 `CoreNetworkModule`（Retrofit + Gson + 6 个 API 提供方法） |

`:core-network` 新增依赖：`retrofit2:converter-gson`、`retrofit2:retrofit`、`gson`

## 五、`:data-repository` 文件归属

### → `:feature-auth`（认证仓库）

| 文件 | 说明 |
|------|------|
| **新建** `AuthRepositoryImpl.kt` | 实现 login/register/logout/sendCode/isLoggedIn |
| `UserRepositoryImpl.kt` 的 auth 部分 | Login/Register 中的 token 保存逻辑 |

依赖：`:core-network`（AccountApi, AuthApi）、`:data-local`（UserLocalDataResource）、`:lib-base`（AuthPreferences, UserAccount）

### → `:feature-mine`（用户仓库）

| 文件 | 说明 |
|------|------|
| **新建** `UserRepositoryImpl.kt` | 实现 getUserProfile/updateProfile/uploadAvatar |
| `UserMappers.kt` | DTO → UserAccount 映射 |

依赖：`:core-network`（UserApi, FileApi）、`:lib-base`（UserAccount）、`:data-repository`（UriFileResolver）

### → `:feature-video`（视频+评论仓库）

| 文件 | 说明 |
|------|------|
| `VideoRepositoryImpl.kt` | 视频仓库实现 |
| `CommentRepositoryImpl.kt` | 评论仓库实现 |
| `VideoMappers.kt` | DTO 映射 |
| `CommentMappers.kt` | DTO 映射 |

依赖：`:core-network`（VideoApi, CommentApi）、`:domain`（Video, Comment 模型）

### → `:lib-base` 或保留共享

| 文件 | 说明 |
|------|------|
| `UriFileResolver.kt` | :feature-auth 和 :feature-mine 都需要（头像上传），迁入 `:lib-base` 或保留在 data-repository 迁移后的某个共享位置。**建议迁入 `:lib-base`** |

## 六、Hilt DI 模块变更

### `:core-network/CoreNetworkModule`（合并后）

```
@Provides OkHttpClient（2 个）
@Provides HttpLoggingInterceptor
@Provides Gson
@Provides Retrofit（@Named("backend")）
@Provides AccountApi, UserApi, FileApi, AuthApi, VideoApi, CommentApi
```

### 各 feature 模块新建 Module

| 模块 | Hilt Module | 提供内容 |
|------|-----------|---------|
| `:feature-auth` | `AuthModule.kt` | `AuthRepositoryImpl → AuthRepository` |
| `:feature-mine` | `MineModule.kt` | `UserRepositoryImpl → UserRepository` |
| `:feature-video` | `VideoModule.kt` | `VideoRepositoryImpl → VideoRepository`, `CommentRepositoryImpl → CommentRepository` |
